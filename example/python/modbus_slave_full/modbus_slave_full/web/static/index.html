💡 工作原理

  • FC20 (读文件记录): 从指定的文件记录位置读取数据，实际读取保持寄存器
  • FC21 (写文件记录): 向指定的文件记录位置写入数据，实际写入保持寄存器
  • 映射关系: 记录编号 = 保持寄存器地址，记录长度 = 读写的寄存器数量
  • 文件编号: 用于逻辑分组，不影响实际地址（本实现中文件编号为0-9）

💡 使用说明

  • 实时生效: 调整后立即生效，无需重启服务器
  • 数据保留: 扩大时保留原有数据，缩小时保留前N个数据
  • 推荐值: 通常100-1000足够使用，最大支持65536
  • 性能考虑: 数据块过大可能影响性能和内存使用
  • 字符串存储: 每2个字符需要1个寄存器
